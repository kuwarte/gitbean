# Lessons

## 2026-07-15

**Java fundamentals**

- Arrays are fixed-size; can't append, must allocate a new array + `System.arraycopy`
- `String` is immutable; use `StringBuilder` for building strings in loops
- `try`-with-resources auto-closes anything implementing `AutoCloseable`
- `MessageDigest.getInstance()` is a factory pattern, not `new`
- `Paths` (plural, utility class with static methods) vs `Path` (interface, no static methods) — easy typo to make
- `System.out.write(byte[])` vs `System.out.println(String)` — `write()` outputs raw bytes with nothing added, needed for content since file bytes might not be valid text
- `.split(" ")` on a header string like `"blob 12"` → `["blob", "12"]`, index into it for type/size
- `return;` inside an `if`-branch exits a method early so later code doesn't run (fallthrough control)
- `Map<String, String>` + `LinkedHashMap` — preserves insertion order, unlike plain `HashMap`, which matters for a readable/predictable index file
- `line.split(" ", 2)` — the `2` limits the split to 2 pieces, so a filename with a space doesn't get mangled

**Git internals**

- Git splits a SHA-1 hash into a 2-char folder + 38-char filename to avoid huge flat directories
- Confirmed: gitbean's `hash-object` output matches `git hash-object` exactly → object format is correct
- Confirmed: real `git cat-file -p` can decompress and read objects written by gitbean's `writeObject` → confirms zlib compression settings match real Git too, not just the hashing
- `add` doesn't touch history — it only updates the index (staging area), a snapshot of what _would_ go into the next commit
- Chose a simplified plain-text index format over real Git's binary format for now, to reach `commit`/`log` sooner — real format documented in `OBJECT-FORMAT.md` for a later upgrade

**Tooling / workflow**

- `mvn exec:java` does **not** auto-recompile — always chain `mvn compile exec:java ...`, or you silently run stale bytecode
- Running gitbean commands directly inside the `gitbean/` project folder pollutes the repo with `.gitbean/` and test files — use a separate sandbox folder instead
- A shell alias (`alias gitbean='java -cp ~/code/gitbean/target/classes com.kuwarte.gitbean.App'`) removes the need to retype the full classpath every test run

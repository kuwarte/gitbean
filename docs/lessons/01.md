# Lessons

## 2026-07-15

- Java arrays are fixed-size; can't append, must allocate new array + System.arraycopy
- String is immutable; StringBuilder for building strings in loops
- try-with-resources auto-closes anything implementing AutoCloseable
- MessageDigest.getInstance() is a factory pattern, not `new`
- Git splits SHA1 hash into 2-char folder + 38-char filename to avoid huge flat dirs
- Confirmed: my hash-object output matches `git hash-object` exactly -> object format is correct
- mvn exec:java does NOT auto-recompile - always chain `mvn compile exec:java ...` or run stale bytecode silently
- Paths (plural, utility class with static methods) vs Path (interface, no static methods) - easy typo
- System.out.write(byte[]) vs System.out.println(String) - write() is for raw bytes, doesn't add anything,
  needed for content since file bytes might not be valid text
- .split(" ") on a header string like "blob 12" -> ["blob", "12"], index into it for type/size
- return; inside an if-branch is used to exit a method early so later code doesn't run (fallthrough control)
- Confirmed: real `git cat-file -p` can decompress and read objects written by gitbean's writeObject -
  confirms zlib compression settings match real Git too, not just the hashing
